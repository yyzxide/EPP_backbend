# 大厂面试级 EPP 终端安全管控平台 (Advanced EPP Backend V2.0 终极版)

## 核心定位
从单纯的“CRUD 管理系统”升级为**“支持十万级长连接、高可靠防丢失、千万级策略秒级下发的高可用分布式中台”**。
不比拼谁用的框架多，而是比拼**对高并发异常场景（网络抖动、节点宕机、惊群效应）的治理深度**。

---

## 🌟 核心护城河：三大高并发深度场景 (BAT 面试必杀技)

### 深度一：十万级长连接 Netty 网关的“惊群效应”与路由一致性
*   **业务痛点**：公司进行机房网络割接，或者某台 Netty 网关节点意外 OOM 宕机，导致挂载在上面的 3 万个 EPP 客户端瞬间断开并同时重连，瞬间的 TCP 建连风暴（惊群效应）直接打垮后端的鉴权服务和数据库。
*   **大厂级解决方案**：
    1.  **客户端退避**：在服务端下发配置，让客户端在断线重连时采用**指数退避算法**（Exponential Backoff，如 1s, 2s, 4s, 8s + 随机 Jitter 抖动），打散重连请求。
    2.  **网关无状态与路由注册**：Netty 本身只做数据包拆粘包和 WebSocket/TCP 维持。设备上线时，将 `DeviceID -> Netty Node IP` 存入 Redis Hash。后台下发控制指令（如解隔离 `deIsolate`）时，根据 Redis 路由表通过 Redis Pub/Sub 或 RocketMQ 投递给特定 Netty 节点。
    3.  **心跳幽灵设备剔除**：Netty 实现 `IdleStateHandler`，结合 Redis 的 TTL 机制。如果客户端 3 次没发心跳，主动断开连接并清除 Redis 路由，防止路由表脏数据。

### 深度二：基于版本号（Sync Time）的千万级策略增量推拉体系
*   **业务背景**：（提取自 C++ 源码 `cStrategyTemplateBridgePlugin.cpp`）管理端修改了全网 10 万台设备的“安全检查基线策略”。如果直接群发完整的 JSON 策略，不仅带宽被打满，客户端解析也慢。
*   **大厂级解决方案**：
    1.  **推拉结合 (Push + Pull)**：后台策略变更后，Netty 不下发完整数据，只通过长连接广播一条极小的信号：“策略已更新，最新 SyncTime = 168439201”。
    2.  **增量同步与多级缓存**：客户端收到信号后，比对本地的 SyncTime，如果不一致，发起 HTTP 请求拉取差异。
    3.  **缓存穿透保护**：由于 10 万台设备同时发起 HTTP Pull 请求，如果直接查 MySQL 会瞬间宕机。引入 **Caffeine (JVM 本地缓存) + Redis (分布式缓存)** 多级缓存架构，保证十万级并发读取策略时，QPS 轻松抗住且 MySQL 零压力。

### 深度三：海量体检日志上报的“幂等性”与“防丢失”设计
*   **业务背景**：（提取自 `cseccheckmgr.cpp` 开机体检）早上 9 点十万台设备开机，瞬间涌入几十万条体检和漏洞日志。网络不好时，客户端可能重发日志。
*   **大厂级解决方案**：
    1.  **Kafka 削峰**：Controller 层收到日志 JSON 后，只做简单格式校验，直接打入 Kafka，迅速返回 `200 OK` 给客户端，保证客户端不卡顿。
    2.  **消费端幂等性防重**：客户端生成唯一 `LogID`（UUID+时间戳）。消费者消费时，利用 **Redis 布隆过滤器 (Bloom Filter)** 或 Redis `SETNX` 判断该 LogID 是否处理过。拒绝重复插入，保证 Elasticsearch / MySQL 里的审计日志绝对准确。
    3.  **批量异步入库优化**：MyBatis-Plus 使用 `saveBatch`，Elasticsearch 使用 `Bulk API`。按时间窗口（如每秒）或批次大小（如 1000 条）攒批入库，大幅降低 IO 开销。

## 🛠 全新架构技术栈 (V2.0)

*   **基础与容器**: Spring Boot 3.x + Java 17 + Docker (模拟多节点)
*   **持久化与缓存**: MySQL 8.0 (基础元数据) + MyBatis-Plus + Redis (路由表、分布式锁、多级缓存、幂等去重)
*   **大数据与搜推**: Kafka (亿级日志削峰填谷) + Elasticsearch (海量安全审计多维检索)
*   **网络与并发**: Netty (C100K 级别自研长连接网关)
*   **AI 应用工程**: Spring AI + DeepSeek API (自然语言查日志、智能安全事件研判)
*   **设计模式应用**:
    *   **策略+工厂模式**：彻底消除日志解析的 `if-else`。
    *   **责任链模式**：用于 Netty 中对客户端数据包的鉴权、解密、心跳处理流水线。
